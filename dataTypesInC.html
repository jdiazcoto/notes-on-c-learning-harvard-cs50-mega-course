Data Types and Variables for the C language

  1. int
    • The int data type is used for variables that will store integers
    • Integers always take up 4 bytes of memory(32 bits). This means the range of values they can store is necessarily limited to 32 bits worth of info
  2. unisgned int
    • unisgned is a qualifier that can be applied to certain types (including int), which effectively doubles the positive range of variables of that ype, at the cost of disallowing any negative values
    • You'll occasionally have use for unisgned variables
  3. char
    • The char data type is used for variables that will store single characters.
    • Characters always take up 1 byte of memory(8 bits). This means the range of values they can store is necessarily limited to 8 bits worth of information.
    • Thanks to ASCII,we've developed a mapping of characters like A,B,C, etc... to numeric values in the positiveside of this range
  4. float
    • The float data type is used for variables that will store floating-point values, also known as real numbers.
    • Floating points values always take up 4 bytes of memory (32 bits)
    • It's a little complicated to describe the range of a float but suffice it to say with 32 bits of precision some of which might be use for an integer part, we are limited in how precise we can be
  5. double
    • The double data type is used for variables that will store floating-point values, also known as real numbers.
    • The difference is that doubles are double precision. They always take up 8 bytes of memory(64 bits)
    • WIth an additional 32 bits of precision relative to a float, doubles allow us to specify much more precise real numbers
  6. void
    • Is a type, but not a data type.
    • Functions cn have a void return type, which just means they don't return a value
    • The parameter list of function can also be void. It simply means the function takes no parameters.
    • For now, think of void more as a placeholder for "nothing". It's more complex than that, but for now that is alright.
